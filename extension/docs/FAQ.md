# 常见问题

## 安装相关

### Q1: 如何安装这个扩展？

1. 下载项目代码
2. 打开 Chrome，地址栏输入 `chrome://extensions/`
3. 右上角开启「开发者模式」
4. 点「加载已解压的扩展程序」
5. 选择 `extension` 文件夹

详见 [README.md](../README.md#-安装)

### Q2: 为什么需要在 Chrome Web Store 之外安装？

当前是开发者版本，适合尝鲜和自定义。后续会发布到 Chrome Web Store，届时可一键安装。

### Q3: 支持 Firefox / Safari 吗？

- **Firefox**：需要改 `manifest.json` 为 MV2 格式（后续支持）
- **Safari**：需要用 Xcode 打包（暂不支持）
- **Edge / Brave**：完全兼容（基于 Chromium）

### Q4: 装了扩展后图标在哪？

- 点 Chrome 工具栏的拼图图标 🧩
- 找到「Web Clipper」
- 点钉子图标 📌 固定到工具栏

## 使用相关

### Q5: 点扩展图标没反应？

**排查步骤：**

1. **检查扩展是否启用**
   - `chrome://extensions/` 确认开关打开

2. **检查错误日志**
   - `chrome://extensions/` 找到本扩展
   - 点「错误」按钮查看
   - 或点「检查视图：Service Worker」看控制台

3. **检查页面是否支持**
   - `chrome://` 内部页面不允许扩展执行
   - Chrome Web Store 页面禁止扩展

4. **重新加载扩展**
   - 点扩展卡片的「刷新」图标 🔄

### Q6: 文件下载到哪里？

浏览器默认下载目录：
- **Windows**: `C:\Users\用户名\Downloads\`
- **macOS**: `/Users/用户名/Downloads/`

按 `Ctrl+J`（macOS: `Cmd+Shift+J`）查看下载记录。

### Q7: 可以修改下载目录吗？

扩展无法修改下载目录（浏览器安全限制）。但你可以：
- 在 Chrome 设置里改默认下载位置：`chrome://settings/downloads`
- 开启「下载前询问每个文件的保存位置」

### Q8: 文件名是"下载"而不是标题？

**已修复**（v1.1.0）。如果还出现：
- 确认装的是最新版
- 重新加载扩展（`chrome://extensions/` 点刷新）
- 提 Issue 报告具体 URL

### Q9: 抓取的内容包含很多无用信息？

工具会自动去除导航、广告、相关推荐等噪音。如果仍有问题：
- 提 Issue 附上测试 URL
- 说明期望去除的内容

### Q10: 能抓取需要登录的页面吗？

**可以**。扩展在当前页面执行，会携带你的登录状态。所以能抓取：
- 已登录的公众号后台
- 已登录的内部系统
- 已登录的付费内容

### Q11: 抓取时会影响原页面吗？

**不会**。扩展克隆 DOM 后操作，不修改原页面。抓取后页面内容不变。

### Q12: 支持 B站、YouTube 字幕吗？

当前版本不支持。后续会考虑增加平台专用解析。

## 功能相关

### Q13: 支持 Markdown 以外的格式吗？

当前只支持 Markdown。后续会考虑：
- HTML 格式（保留原始样式）
- 纯文本

### Q14: 能选中部分文字保存吗？

当前只支持整页抓取。后续会增加「选中文字保存为片段」功能（右键菜单）。

### Q15: 能批量保存页面图片吗？

暂不支持。后续会增加「批量保存图片」功能。

### Q16: 有历史记录吗？

暂无。所有抓取直接下载到本地，不保存历史。后续可能增加。

## 权限相关

### Q17: 为什么需要这些权限？

| 权限 | 用途 |
|------|------|
| `activeTab` | 访问当前标签页 |
| `contextMenus` | 添加右键菜单 |
| `scripting` | 注入脚本提取内容 |
| `host_permissions` | 在任意网页执行 |

详见 [ARCHITECTURE.md](ARCHITECTURE.md#-权限说明)

### Q18: 会收集我的数据吗？

**不会**。本扩展：
- 不收集任何用户数据
- 不发送数据到服务器
- 所有处理在本地完成
- 不需要 Google 账号

### Q19: 代码开源吗？
