# 测试指南

本文档说明如何测试 Web Clipper 扩展。

## 🧪 测试环境

- Chrome 88+ 或 Edge 88+
- 已加载扩展（开发者模式）

## 📋 测试清单

### 基础功能测试

#### T1: 扩展加载
- [ ] `chrome://extensions/` 开启开发者模式
- [ ] 加载 `extension` 文件夹
- [ ] 扩展列表显示「Web Clipper - 智能抓取」
- [ ] 无错误提示

#### T2: 图标点击抓取
- [ ] 打开 `https://example.com`
- [ ] 点工具栏扩展图标
- [ ] 文件下载到浏览器下载目录
- [ ] 文件名：`imported_Example_Domain.md`
- [ ] 文件内容包含标题和正文

#### T3: 右键菜单抓取
- [ ] 在 `https://example.com` 右键
- [ ] 菜单显示「📥 智能抓取此页」
- [ ] 点击后文件下载
- [ ] 文件内容正确

### 平台兼容性测试

#### T4: 微信公众号
- [ ] 打开真实公众号文章
- [ ] 点扩展图标
- [ ] 文件名是文章标题（不是"下载"）
- [ ] 文件内容包含正文
- [ ] 无导航/广告噪音

#### T5: 博客类网页
- [ ] 打开任意博客文章（如 WordPress）
- [ ] 抓取成功
- [ ] 正文完整

#### T6: 新闻类网页
- [ ] 打开新闻网站文章页
- [ ] 抓取成功
- [ ] 正文无相关推荐

### 边界情况测试

#### T7: 空标题页面
- [ ] 打开标题为空的页面
- [ ] 文件名用域名兜底
- [ ] 不报错

#### T8: 超长标题
- [ ] 打开标题很长的页面
- [ ] 文件名截断到 50 字符
- [ ] 不报错

#### T9: 特殊字符标题
- [ ] 打开标题含 `<>:"/\|?*` 的页面
- [ ] 特殊字符替换为 `_`
- [ ] 文件名合法

#### T10: 动态加载页面
- [ ] 打开 SPA 应用
- [ ] 等页面加载完成再抓取
- [ ] 正文提取正确

### 内容质量测试

#### T11: 噪音去除
- [ ] 抓取的 md 无 `<script>` 标签
- [ ] 无 `<style>` 标签
- [ ] 无导航链接
- [ ] 无广告文字
- [ ] 无相关推荐
- [ ] 无评论区内容

#### T12: 段落格式
- [ ] `<h1>` 转成 `#`
- [ ] `<h2>` 转成 `##`
- [ ] `<p>` 之间有空行
- [ ] `<pre>` 转成代码块
- [ ] `<blockquote>` 转成引用
- [ ] `<ul>/<ol>` 转成列表

#### T13: frontmatter
- [ ] 文件开头有 `---`
- [ ] 包含 title、source、domain、imported 字段
- [ ] 格式正确

## 🐛 Bug 报告

测试发现问题，按以下格式报告：

```
**测试用例**：T4 微信公众号
**测试 URL**：https://mp.weixin.qq.com/s/xxx
**期望结果**：文件名是文章标题
**实际结果**：文件名是"下载"
**环境**：Chrome 149.0 / Windows 11
**截图**：（如适用）
```
